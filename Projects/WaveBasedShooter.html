System Functionality

For this course I delivered, the end goal was to create a working First Person Shooter project that works very similialry to that of Treyarch Call Of Duty Zombies mode, where infinite waves of spawning artiticla intelligence will chase and attack the player and you have to surivive for as long as possible. For this game project I was delivering I broke down the course into these section until the learners were able to build a fully functional protoype. These were:

  • Creating the Gun Super Class
  • Creating virtual functions for Gun class
  • Importing and configriung Skeletal Meshes
  • Bone Socketing For Weapon Attach System
  • Finite & Behaviour Tree Artifical Intelligence
  • Dynamic Zone AI Spawning System

For the spawning of artifical intellgence instead of utilsing a random spawn location in range system. Instead I took insipiration from Call of Duty & Killing Floor for how their artifical intelligence is spawned within the world. From my obersvations I conclude that these games may use a zone based system for determining what active zones the player is currently standing in.

Zone Based Spawning System

The zone based system i devised works by having hidden colliders within a 3D enviornment with each collider there are spawner point actor in an array located on the colliders class. Once the player steps into these zones then my game mode will determine which spawning points are currently 'active' then using random number generation the spawning system will choose of the active locations to make a dynamic system.

To teach my learners about performance this spawning system has restriction in place, these retriscrtion are that the AI cannot spawn past that max quaintiy by a variable limit. This ensures that there will not be to many calls on the CPU and GPU with extensive amounts of usage and calls.

Inverse Kinematics System

For the weapon system in order to save on development time, instead of creating additonal animations per weapon. For this I devised an Inverse Kinematics system so that the left hand bone of the players skeletal mesh would be attacthed to a custom bone socket per weapon skeleton. As seen in the preview by simply moving the position of said bone in the weapon skeleton this will dynamically adjust the skeltal mesh to better snap and position for each weapon animation set.
